## 🔍 详细分析

### 1. 主文本颜色差异

**发现**：参考网页在不同页面使用了略有不同的主文本颜色

- **首页主文本**：`#333333` (rgb(51, 51, 51))
  - 当前项目 `textPrimary: '#333333'` ✅ **完全匹配**

- **文章页主文本**：`#303133` (rgb(48, 49, 51))
  - 与 `#333333` 的差异：
    - Red: 51 → 48 (减少 3)
    - Green: 51 → 49 (减少 2)
    - Blue: 51 → 51 (不变)
  - **视觉差异极小**，人眼几乎无法区分

**建议**：保持当前的 `#333333`，与首页完全一致即可

---

### 2. 次要文本颜色 ⚠️ 重要

**参考网页（首页）**：`#808080` (rgb(128, 128, 128))
**当前项目**：`#666666` (rgb(102, 102, 102))

**差异分析**：
- `#666666` 更深、对比度更高
- `#808080` 更浅、更柔和

**视觉对比**：
```
#666666 ████████  当前项目（较深）
#808080 ████████  参考网页（较浅）
```

**建议**：
- 🔴 **建议调整为 `#808080`**
- 理由：参考网页使用更浅的灰色，视觉层次更柔和
- 影响：次要文本会更轻盈，不会抢主文本的视觉焦点

---

### 3. 第三层文本颜色 ✅

**参考网页（文章页）**：`#999999` (rgb(153, 153, 153))
**当前项目**：`#999999`

✅ **完全匹配**！

## 📋 改进建议

### 高优先级 🔴

**调整 `textSecondary` 颜色**
```typescript
// 当前
textSecondary: '#666666',  // rgb(102, 102, 102)

// 建议调整为
textSecondary: '#808080',  // rgb(128, 128, 128)
```

**理由**：
1. 匹配参考网页首页
2. 视觉层次更柔和
3. `#666` 与 `#333` 对比不够明显，容易混淆层级

### 中优先级 🟡

保持 `textPrimary: '#333333'`，不需要调整为文章页的 `#303133`
- 差异极小（人眼不可见）
- 与首页完全一致
- 代码维护更简单
